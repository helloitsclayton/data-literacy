Data Lifecycles

This is an example from the world of libraries, and represents a data lifecycle in which the results of research aren't necessarily being published or made public. An entity (in this case a library) collects data on library users based on certain activities, stores that (usually anonymized) data, and provides access to those who are supposed to be analyzing it. The analysis generates reports, for internal or external use, the entity decides what data should be retained and purges the rest.
This is a representation of the data lifecycle in a more traditional academic research context. Here, researchers plan/design a study, collect data and create a framework for systematically storing and analyzing that data. Researchers then evaluate the results of analysis and archive their data so others can use it (sometimes!). Then they publish research based on that data and possibly provide a means of accessing the data so it can be reused.

Let's take a slightly different view of that cycle, focusing specifically on the data analysis process for projects that aren't generating their own data:
  1. Come up with a question
  2. Find existing data related to question
  3. Explore and clean data to support analysis
  4. Analyze data (probably with stats)
  5. Present data (publication, repositories, etc.)
